# 交易日志代写任务书

你在替一位 A 股短线交易者写他自己的交易日志(复盘笔记)。他的策略:每天开盘后 9:39 由量价模型
(V16,LightGBM 排序)从热门板块里选出 Top 榜单,买入榜单头部的票;持仓原则 T+2 卖出,但操作弹性:
盘面不佳/抛压明显会提前卖,当天技术面特别好会多拿一两天。

## 你的输入

1. 「事实包 JSON」——这笔交易的全部客观数据(附在任务末尾)。
2. 「范文」——他之前亲手写的日志,**风格、结构、口吻都要仿照范文**(附在任务末尾)。

## 输出要求

把结果 JSON 用 Shell 写入任务指定的结果文件(整个文件就是一个 JSON 对象,UTF-8,无 BOM):

```
{"content": "<对内正文>", "content_external": "<对外正文>"}
```

- `content_external`(对外):完整段落,严格仿范文结构。
- `content`(对内):1~3 句,更坦率的自我复盘口吻(可以说"不够果断""属于博弈"这类真心话)。
- 简体中文;不要 emoji;不要 markdown 标题;数字保留事实包原始精度即可。
- **禁止任何英文缩写/模型代号/系统内部词**(如 LGB、rank、scan、FIFO 之类)——他手写日志
  从不用这些词。评分就写「评分」,排名就写「算法推荐第 N / 策略评分排名第 N」,
  只用范文里出现过的说法。

## 硬性规则(违反=作废)

1. **只准引用事实包里出现的数字**(价格/数量/费用/百分比/时间/排名/成交量额)。一个数都不许编。
   事实包里没有的信息就不写(比如没有过户费就不提过户费;缺分钟线就不写逐分钟走势)。
2. 量额换算:volume_shares → 万股(除以 1e4);amount_yuan → 亿元(除以 1e8,不足 1 亿用 万元)。
   涨跌幅自己算的必须能从事实包数字推出来(如高开幅度 = (开盘-昨收)/昨收)。
3. **买入**对外结构(仿范文):开盘价(高开/平开/低开 ±x.xx%) → 9:30-9:40 早盘成交量(万股)与
   成交额 → 「早盘判断信息后于 h:mm 介入交易」(用事实包的成交时刻) → 买入价格/均价 → 佣金
   (有过户费才写) → 策略叙事:当日算法推荐第 N(v16_scan.my_rank)、板块、CCI14 读数与含义、
   早盘分钟走势结构(用 early_bars 描述回踩/放量/拉升,时间要对得上 bar)
   → 收尾:T+2 预期 / 仓位说明(若 same_day_other_buys 非空可提"当日双票分仓")。
4. **卖出**对外结构(仿范文):先给卖出理由——必须与数据自洽:
   - holding_trade_days_T_plus == 2:「持有到达预期 T+2」+ 结合当日分时(intraday)/持仓期间表现
     (holding_daily)/大盘(index_daily_shanghai)补一句为什么此刻出。
   - < 2(提前卖):从数据找理由——持仓期间个股走弱、当日冲高回落、大盘转弱、抛压明显等,
     写清"提前"的动机(止损/落袋/风险收益不成比例)。
   - > 2(多拿):解释为什么超过 T+2 还持有(前一日技术面强/涨停/量能健康),以及今天为什么出。
   - 若 same_day_sell_legs 非空,这是分批出货中的一腿,措辞体现分批(如「继续减仓」「清仓尾腿」)。
   然后:卖出平均成交价格 → 佣金 → 印花税 → 「清仓收益 X,收益率 Y%」(用 computed_pnl 的
   net_after_fees 和 return_pct;亏损为负照写,叙事按止损/认错口吻)。
   若没有 computed_pnl(历史买入腿不在记录里),收益那句整个不写。
5. **衔接他自己写过的记录**:事实包若带 `your_buy_note`(他手写的买入记录)或
   `your_same_day_sell_notes`(他手写的同日其他卖出腿),写卖出时要与其口吻/口径衔接
   (比如他买入时说了「试验仓」「留 40% 仓位」,卖出就顺着这个背景写)。
   但**卖出理由独立判断**——卖早了的原因(盘面转弱/抛压/落袋)可以与买入理由完全无关,
   不要硬扯买入叙事。他写的内容只是背景,绝不复述式照抄。
6. 可以用 SearchWeb 查这只票在交易日前后的题材/板块消息,给策略叙事补一句背景。
   查不到就不写题材,不许编新闻。
7. 写完只输出结果文件,不需要其他解释。
